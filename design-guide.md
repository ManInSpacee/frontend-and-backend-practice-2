# Brutalist / Terminal Design Guide

## Основная идея

Этот стиль имитирует интерфейсы программистов — терминалы, IDE, системные утилиты.
Он говорит: "это инструмент, а не украшение". Пользователь чувствует себя в контроле.

---

## 5 правил которые нельзя нарушать

### 1. Никаких скруглений
`border-radius: 0` везде. Абсолютно везде.
Скругления делают интерфейс "мягким" и "дружелюбным" — это противоречит стилю.

```css
/* Плохо */
.button { border-radius: 8px; }

/* Хорошо */
.button { border-radius: 0; }
```

### 2. Один акцентный цвет
Выбери один яркий цвет и используй его только для важных вещей:
- активная ссылка
- кнопка подтверждения
- выделение цены
- иконка бренда

Больше одного акцентного цвета — хаос.

```css
--accent: #ff4d00;  /* и только он */
```

### 3. Монospace шрифт
`Courier New` или `JetBrains Mono` или `Fira Code`.
Это важно — обычный шрифт ломает атмосферу.

```css
font-family: 'Courier New', Courier, monospace;
```

### 4. Границы вместо теней
Тени (`box-shadow`) размывают края и создают "мягкость". Вместо них — чёткая граница.

```css
/* Плохо */
.card { box-shadow: 0 4px 20px rgba(0,0,0,0.3); }

/* Хорошо */
.card { border: 1px solid #2a2a2a; }

/* Hover — усиливаем границу */
.card:hover { border-color: #444; }
```

### 5. Uppercase + letter-spacing для текстовых меток
Все лейблы, заголовки разделов, кнопки — в верхнем регистре с трекингом.

```css
.label {
  text-transform: uppercase;
  letter-spacing: 1px;
  font-size: 12px;
}
```

## Цветовая схема

```
#0a0a0a  — фон страницы       (не чисто чёрный — слишком агрессивно)
#111111  — фон карточек
#1a1a1a  — hover состояние
#2a2a2a  — граница обычная
#444444  — граница при фокусе
#666666  — приглушённый текст
#e0e0e0  — основной текст     (не белый — слишком контрастно для монитора)
#ff4d00  — акцент             (меняй на свой вкус)
```

Замена акцента под настроение:
```
#00ff88  — зелёный  (хакер, матрица)
#00aaff  — синий    (холодный tech)
#cc00ff  — фиолет   (cyberpunk)
#ffcc00  — жёлтый   (предупреждение, осторожность)
```

## Структура типичного компонента

### Карточка
```html
<div class="card card-accent">
  <span class="label">Категория</span>
  <h3>Заголовок</h3>
  <p class="text-muted">Описание</p>
  <div style="margin-top: 16px; display: flex; justify-content: space-between">
    <span class="text-accent">9 990 ₽</span>
    <span class="label">15 шт.</span>
  </div>
</div>
```

### Кнопки — иерархия
```html
<!-- Одно главное действие на странице -->
<button class="btn-primary">Подтвердить</button>

<!-- Второстепенное действие -->
<button class="btn-secondary">Отмена</button>

<!-- Опасное действие — только там где реально опасно -->
<button class="btn-danger">Удалить</button>

<!-- Ссылка-действие без визуального веса -->
<button class="btn-ghost">Подробнее</button>
```

### Статус
```html
<span class="badge badge-success">Активен</span>
<span class="badge badge-danger">Заблокирован</span>
<span class="badge badge-accent">Администратор</span>
```

## Как добавить в проект

1. Скопируй `design-system.css` в папку проекта
2. Подключи в `main.jsx` или `index.html`:
```js
import './design-system.css';
```
3. Настрой переменные под проект — смени `--accent`, при желании замени `--font`
4. Используй классы из файла напрямую

---

## Чего избегать

| Нельзя | Почему |
|--------|--------|
| `border-radius` больше 0 | Ломает стиль |
| Больше одного акцентного цвета | Хаос, нет фокуса |
| `box-shadow` | Размывает чёткость |
| Обычный шрифт (Arial, Roboto) | Теряется атмосфера |
| Анимации дольше 200ms | Стиль про скорость и точность |
| Светлый фон | Стиль держится на тёмной теме |
| Градиенты | Слишком декоративно |
